# 项目说明

## 项目是什么

本项目是一个面向百度 SEO 的淘客/导购内容网站，核心目标是通过稳定、可索引、结构清晰的内容页面承接搜索流量，并把用户引导到淘宝、天猫等外部平台完成购买。

网站形态类似轻量版“什么值得买”，但第一阶段不做社区、不做复杂用户体系、不做返利闭环，只聚焦内容生产、商品展示、优惠信息聚合和基础后台管理。

第一阶段的主要页面包括：

- 导购文章页：用于承接长尾关键词，例如选购指南、清单推荐、横评、优惠解读。
- 商品详情页：展示商品基础信息、价格、优惠券、购买链接和导购说明。
- 分类页：按品类聚合文章和商品，形成稳定的 SEO 入口。
- 专题页：围绕活动、场景、人群或品类做内容聚合。
- 简单后台：支持内容、商品、分类、专题的基础维护。

## 项目不是什么

本项目第一阶段不是商城，也不是返利平台。

明确不做：

- 不做站内交易。
- 不做购物车。
- 不做订单支付。
- 不做用户提现。
- 不做返利系统。
- 不做用户分销。
- 不做多级代理。
- 不做复杂会员体系。
- 不做 App。
- 不做实时比价平台。
- 不做内容社区和评论互动。

所有交易行为都发生在外部电商平台。本网站只负责导购内容展示、商品信息整理、优惠入口承接和跳转。

## 备案和运营边界

项目主体按个体户备案思路设计，第一阶段应避免引入需要更高合规成本的业务形态。

开发时需要遵守以下边界：

- 页面文案不承诺返利到账。
- 不展示“站内购买”“立即支付”等容易造成交易误解的表述。
- 外部跳转按钮应表达为“去淘宝查看”“领券购买”“查看优惠”等。
- 商品价格、券信息、佣金、库存等来自外部接口的数据需要标注更新时间或用弱承诺表达。
- 不在站内沉淀用户订单和支付信息。

## 第一阶段目标

第一阶段目标是做出一个能持续发布内容、被搜索引擎抓取、可手动维护商品和专题的 MVP。

具体目标：

- 建立清晰的信息架构：文章、商品、分类、专题之间关系稳定。
- 支持 SEO 基础能力：服务端渲染、静态元信息、规范 URL、面包屑、站点地图、robots。
- 支持导购内容生产：后台可以创建、编辑、发布文章。
- 支持商品信息维护：后台可以维护商品标题、图片、价格、优惠券、外部链接。
- 支持专题聚合：后台可以把文章和商品组合成活动页或场景页。
- 支持后期接口接入：所有外部淘客接口先通过 adapter 抽象，不把真实接口耦合进业务页面。

## 成功标准

第一阶段完成后，应达到以下状态：

- 网站首页、分类页、文章页、商品页、专题页可以正常访问。
- 页面 HTML 对搜索引擎友好，主要内容不依赖客户端异步加载后才出现。
- 后台可以完成最基本的内容和商品维护。
- 本地环境可以通过 Docker Compose 启动数据库、Redis 和应用依赖。
- 代码结构为后续接入轻淘客、淘宝联盟、商品转链、优惠券接口留出位置。
- 不包含超出 MVP 的返利、支付、订单和分销逻辑。
