# MVP 范围

## 第一阶段只做什么

第一阶段只做支撑 SEO 内容站上线所需的最小功能集合。所有功能都围绕“发布导购内容、展示商品信息、形成可索引页面、便于后台维护”展开。

## 前台页面

### 首页

首页用于展示网站的核心内容入口。

需要包含：

- 推荐文章列表。
- 热门商品列表。
- 主要分类入口。
- 专题入口。
- 基础 SEO meta 信息。

暂不需要：

- 个性化推荐。
- 千人千面。
- 用户信息流。
- 实时热榜算法。

### 文章页

文章页是主要 SEO 承接页面。

需要包含：

- 标题、摘要、正文、封面图。
- 发布时间和更新时间。
- 所属分类。
- 关联商品。
- 面包屑导航。
- SEO title、description、canonical。
- 可由后台控制发布状态。

暂不需要：

- 评论。
- 点赞。
- 收藏。
- 打赏。
- 复杂富文本协同编辑。

### 商品页

商品页用于承接商品词、优惠词和长尾搜索。

需要包含：

- 商品标题。
- 商品主图。
- 商品简介。
- 当前价格。
- 原价或参考价。
- 优惠券信息。
- 外部购买链接。
- 来源平台字段。
- 更新时间。
- 关联文章。
- SEO title、description、canonical。

暂不需要：

- 站内购买。
- 库存锁定。
- 购物车。
- 商品 SKU 选择。
- 价格历史曲线。
- 自动比价。

### 分类页

分类页用于形成稳定栏目结构。

需要包含：

- 分类名称、描述。
- 分类下文章列表。
- 分类下商品列表。
- 分页。
- SEO meta 信息。

暂不需要：

- 复杂筛选器。
- 多维排序。
- 用户自定义订阅。

### 专题页

专题页用于承接活动、场景、人群、品牌或品类聚合。

需要包含：

- 专题标题、描述、封面图。
- 专题正文或导语。
- 关联文章列表。
- 关联商品列表。
- 发布状态。
- SEO meta 信息。

暂不需要：

- 可视化搭建器。
- 拖拽装修。
- A/B 测试。
- 千人千面专题。

## 后台功能

后台第一阶段只服务内容维护，不做运营大屏。

需要包含：

- 登录保护，可以先使用单管理员账号方案。
- 文章管理：创建、编辑、发布、下线、列表查询。
- 商品管理：创建、编辑、发布、下线、列表查询。
- 分类管理：创建、编辑、排序、启用、停用。
- 专题管理：创建、编辑、发布、下线。
- 文章与商品关联。
- 专题与文章、商品关联。

暂不需要：

- 多角色权限。
- 审核流。
- 操作日志审计。
- 数据看板。
- 内容自动采集。
- AI 自动写稿。

## SEO 基础能力

第一阶段必须考虑：

- 每个可收录页面有唯一、稳定 URL。
- 支持自定义 slug。
- 支持 title、description。
- 支持 canonical。
- 支持 robots.txt。
- 支持 sitemap.xml。
- 支持服务端渲染或静态生成主要内容。
- 支持 404 页面。
- 页面内使用合理的 H1、H2、面包屑和内链。

第一阶段暂不做：

- 大规模程序化 SEO 页面生成。
- 自动关键词库。
- 自动采集和伪原创。
- 多站群。

## 外部接口

第一阶段只预留接口抽象，不直接接真实接口。

需要预留：

- 轻淘客接口 adapter。
- 淘宝联盟/阿里妈妈 adapter。
- 商品转链 adapter。
- 优惠券查询 adapter。
- 订单查询 adapter 的扩展位置。

第一阶段实现方式：

- 使用 mock adapter 返回固定结构数据。
- 业务代码只依赖内部接口类型，不依赖第三方 SDK 或具体 HTTP 字段。
- 外部接口密钥只允许从环境变量读取。

暂不需要：

- 真实接口联调。
- 自动转链任务。
- 订单同步。
- 佣金结算。
- 接口失败重试队列。

## 明确不做的功能

以下功能不进入 MVP：

- 返利系统。
- 用户提现。
- 用户分销。
- 多级推广。
- 站内交易。
- 支付。
- 购物车。
- 订单中心。
- App。
- 小程序。
- 社区评论。
- 用户私信。
- 积分系统。
- 复杂会员体系。
- 自动采集系统。
- 站群管理。

如果后续任务涉及以上功能，必须先确认是否进入新阶段，不能在第一阶段顺手实现。
